# 软件需求说明书 (SRS) — 简易 Web 计算器 (simple-web-calculator)

## 1. 系统边界与概述 (System Boundary & Overview)
- **业务概述**：设计并实现一个运行在 Web 端的简易计算器，兼顾美观性与良好的跨平台适配性，满足用户日常的快捷计算需求。
- **系统边界**：
  - **在册范围**：
    - 计算器核心计算引擎：基础加减乘除、清除（C/AC）、退格（Backspace）、百分比、正负号切换、小数点输入。
    - 响应式网页界面：自适应 PC、平板及手机等不同分辨率的屏幕。
    - 视觉美化与交互体验：提供悬停与点击动效，支持明暗（Dark & Light）双色主题切换。
    - 计算安全与边界控制：浮点精度校正、除零溢出校验、输入字符长度限制。
  - **排除范围**：
    - 科学计算器功能：如三角函数、对数、幂运算等高级函数。
    - 历史计算记录的云端同步或数据库持久化存储（本需求仅支持前端内存态或本地会话态的历史查看）。
    - 第三方账户体系或社交分享功能。

## 2. 数据流拓扑 (Data Flow Topology)
- **数据流描述**：
  计算器接收用户的按键输入（数字/运算符），触发相应的数据模型校验，存入内部状态机（操作数1、运算符、操作数2）。触发 `=` 时执行运算引擎处理，校正精度，并最终输出更新至前端显示屏。若遇到异常（如除零、溢出），显示屏直接输出异常状态词并执行状态锁保护。
- **核心数据流向图/流程**：
```

  [用户按键输入] -> (输入有效性校验/长度限制) -> [更新内存操作数与运算符]
       |
       v (触发 "=" 计算或更改运算符)
  [执行计算引擎] -> (判断除数是否为零) 
       |---> [是] -> [显示 "Error"，锁定输入，仅允许 AC 清除]
       |---> [否] -> (执行高精度纠错 toFixed(10)) -> [更新显示结果，更新当前操作数]

```

## 3. 功能需求定义 (Functional Requirements)

### 3.1 计算引擎模块 (CALC)

#### REQ-CALC-001: 基础四则运算
- **优先级**：高
- **前置条件**：系统加载成功，用户已输入第一个操作数和运算符，且正在输入第二个操作数。
- **业务逻辑**：
  1. 用户点击数字或小数点，构造 `operand1`。
  2. 用户点击运算符（`+`, `-`, `*`, `/`），将状态机的当前运算符设为所选符，并将输入焦点切换至 `operand2`。
  3. 用户点击 `=` 或另一个运算符时，系统根据当前的两个操作数与运算符计算出结果 `result`。
  4. 若计算结果为有效数值，将 `result` 更新为新的 `operand1`，并清空 `operand2` 与运算符，以便进行链式计算。
- **输入输出数据流描述**：
  - **输入**：`operand1` (数字), `operator` (字符 `+`, `-`, `*`, `/`), `operand2` (数字)。
  - **内部处理**：根据 `operator` 对 `operand1` 和 `operand2` 执行标准的数学运算。
  - **输出**：计算结果 `result`，并在界面显示区呈现。

#### REQ-CALC-002: 清除与撤销机制 (C / AC / Backspace)
- **优先级**：高
- **前置条件**：系统加载成功，界面存在输入内容。
- **业务逻辑**：
  1. **清除当前输入 (C)**：重置当前正在输入的数值（如 `operand2` 或当前的 `operand1`），不影响已输入的运算符及另一个操作数。
  2. **全部清除 (AC)**：将整个计算器的状态机重置为初始状态（`operand1 = 0`, `operand2 = null`, `operator = null`）。
  3. **退格撤销 (DEL / Backspace)**：移除当前正在输入的数值的最后一个字符。若当前数值长度被清减为 0，则该数值重置为 `0`。
- **输入输出数据流描述**：
  - **输入**：清除/重置/退格指令。
  - **内部处理**：重置内存中对应的状态变量或对当前操作数执行字符串切片（`slice(0, -1)`）。
  - **输出**：更新界面显示区的值（如重置为 `0` 或缩减后的数字字符串）。

#### REQ-CALC-003: 辅助运算功能 (Decimal / Percent / Negation)
- **优先级**：中
- **前置条件**：系统加载成功。
- **业务逻辑**：
  1. **小数点输入 (.)**：在当前操作数末尾添加小数点。若当前操作数已存在小数点，则此按键无效（防止输入如 `1.2.3`）。若当前未输入任何数字时点击 `.`, 则自动补全为 `0.`。
  2. **百分比转换 (%)**：将当前操作数的值除以 100（即 `val = val / 100`）。
  3. **正负号切换 (+/-)**：反转当前操作数的正负符号（即 `val = -val`）。
- **输入输出数据流描述**：
  - **输入**：小数点/百分比/正负号按键指令。
  - **内部处理**：对当前操作数（字符串或数值）进行正则校验或数值转换。
  - **输出**：更新界面显示。

#### REQ-CALC-004: 边界与异常处理规则
- **优先级**：高
- **前置条件**：计算动作被触发。
- **业务逻辑**：
  1. **除零防护**：若运算符为 `/` 且 `operand2` 的值为 `0`，当触发 `=` 或链式计算时，系统不可崩溃，必须在显示区输出 `Error` 字符。此时锁定除 `AC` 键外的所有按键，直至用户执行 `AC` 键复位。
  2. **高精度纠错**：使用高精度处理（例如运算后先通过 `toFixed(10)` 转换，再使用 `parseFloat` 解析），以解决 JavaScript 语言原生浮点数计算精度丢失（如 `0.1 + 0.2 === 0.30000000000000004`）的问题。
  3. **输入长度限制**：限制单次操作数的最大长度为 12 位字符。达到限制后，任何数字或小数点输入指令均被忽略。
- **输入输出数据流描述**：
  - **输入**：计算操作数及运算符。
  - **内部处理**：判断除数是否为零、进行浮点计算后截断纠偏、校验输入字符累加长度。
  - **输出**：格式化后的运算数值（如最多保留 10 位有效小数）或 `Error` 状态文本。

### 3.2 界面与适配模块 (UI)

#### REQ-UI-001: 响应式跨平台适配
- **优先级**：高
- **前置条件**：无。
- **业务逻辑**：
  1. 采用 CSS Grid (网格布局) 设计计算器键盘，确保所有按键分布均匀。
  2. 针对 PC、平板和手机使用媒体查询 (Media Queries) 实施响应式布局适配：
     - **PC端 (宽屏, >768px)**：计算器渲染为独立的居中浮动卡片（最大宽度限制为 400px），并配有现代化阴影和圆角效果。
     - **平板端 (中屏, 480px 至 768px)**：居中卡片布局，按键触控面积按比例放大，保证舒适的操作体验。
     - **手机端 (窄屏, <480px)**：计算器占满手机屏幕高度与宽度（全屏模式），按键触控面积符合移动端触控标准（建议不小于 44px * 44px）。
- **输入输出数据流描述**：
  - **输入**：客户端设备屏幕分辨率信息。
  - **内部处理**：CSS 媒体查询按预设断点动态应用不同的样式类（卡片布局与全屏布局的自动切换）。
  - **输出**：完美的响应式视觉渲染，无任何内容溢出或按钮错位。

#### REQ-UI-002: 现代美学设计与明暗主题切换
- **优先级**：中
- **前置条件**：系统运行中。
- **业务逻辑**：
  1. 采用现代极简主义设计风格，为数字键、特殊操作符键 and 运算结果显示区设计不同的配色体系，确保对比度高且易于识别。
  2. 提供主题切换控件。支持一键在 **Light Mode (日间明亮主题)** 与 **Dark Mode (夜间暗色主题)** 之间进行流畅过渡。
  3. 为所有按键实现悬停（Hover）及点击（Active）的背景微色变和轻微缩放过渡动效，提升交互的实物感。
- **输入输出数据流描述**：
  - **输入**：主题切换开关触发事件。
  - **内部处理**：动态更新 HTML 标签的 `data-theme` 属性值或改变全局 CSS 变量。
  - **输出**：计算器背景色、按钮底色、文字颜色和阴影颜色在 300ms 内流畅渐变完成。

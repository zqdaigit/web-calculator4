# 软件需求说明书 (SRS) — 简易 Web 计算器 (simple-web-calculator)

## 1. 系统边界与概述 (System Boundary & Overview)
- **业务概述**：设计并实现一个运行在 Web 端的简易计算器，兼顾美观性与良好的跨平台适配性，满足用户日常的快捷计算需求。
- **系统边界**：
  - **在册范围**：
    - 计算器核心计算引擎：基础加减乘除、清除（C/AC）、退格（Backspace）、百分比、正负号切换、小数点输入、平方运算（x²）、开方运算（√）。
    - 响应式网页界面：自适应 PC、平板及手机等不同分辨率的屏幕。
    - 视觉美化与交互体验：提供悬停与点击动效，支持明暗（Dark & Light）双色主题切换。
    - 计算安全与边界控制：浮点精度校正、除零溢出校验、数值溢出控制、输入字符长度限制。
    - 辅助实用功能：会话态历史记录管理、物理键盘快捷键映射及快捷键说明、结果复制与 Toast 提示。
  - **排除范围**：
    - 科学计算器功能：如三角函数、对数、除平方和开方外的其他幂运算等高级函数。
    - 历史计算记录的云端同步或数据库持久化存储（本需求仅支持前端会话态的历史查看）。
    - 第三方账户体系或社交分享功能。

## 2. 数据流拓扑 (Data Flow Topology)
- **数据流描述**：
  计算器接收用户的按键输入（数字/运算符/单值科学计算键），触发相应的数据模型校验，存入内部状态机（操作数1、运算符、操作数2）。触发 `=` 或单值科学计算键时执行运算引擎处理，校正精度，并最终输出更新至前端显示屏。若遇到异常（如除零、安全整数溢出、负数开方），显示屏直接输出异常状态词并执行状态锁保护。计算成功的结果会同步写入 Session Storage 历史记录中，并可通过点击回填。
- **核心数据流向图/流程**：
```

  [用户按键输入/键盘快捷键] -> (输入有效性校验/长度限制) -> [更新内存操作数与运算符]
       |
       v (触发 "=" 计算或更改运算符)
  [执行计算引擎] -> (判断除数是否为零/是否安全整数溢出) 
       |---> [是] -> [显示 "Error" / "Error: Overflow"，锁定输入，仅允许 AC 清除]
       |---> [否] -> (值绝对值 >= 1e15 ? 转换科学计数法 : 执行高精度纠偏 toFixed(10)) 
                          |
                          v
                    [更新显示结果，更新当前操作数] -> [写入 Session Storage 历史记录]

```

## 3. 功能需求定义 (Functional Requirements)

### 3.1 计算引擎模块 (CALC)

#### REQ-CALC-001: 基础四则运算
- **优先级**：高
- **前置条件**：系统加载成功，用户已输入第一个操作数和运算符，且正在输入第二个操作数。
- **业务逻辑**：
  1. 用户点击数字或小数点，构造 `operand1`。
  2. 用户点击运算符（`+`, `-`, `*`, `/`），将状态机的当前运算符设为所选符，并将输入焦点切换至 `operand2`。
  3. 用户点击 `=` 或另一个运算符时，系统根据当前的两个操作数与运算符计算出结果 `result`。
  4. 若计算结果为有效数值，将 `result` 更新为新的 `operand1`，并清空 `operand2` 与运算符，以便进行链式计算。
- **输入输出数据流描述**：
  - **输入**：`operand1` (数字), `operator` (字符 `+`, `-`, `*`, `/`), `operand2` (数字)。
  - **内部处理**：根据 `operator` 对 `operand1` 和 `operand2` 执行标准的数学运算。
  - **输出**：计算结果 `result`，并在界面显示区呈现。

#### REQ-CALC-002: 清除与撤销机制 (C / AC / Backspace)
- **优先级**：高
- **前置条件**：系统加载成功，界面存在输入内容。
- **业务逻辑**：
  1. **清除当前输入 (C)**：重置当前正在输入的数值（如 `operand2` 或当前的 `operand1`），不影响已输入的运算符及另一个操作数。
  2. **全部清除 (AC)**：将整个计算器的状态机重置为初始状态（`operand1 = 0`, `operand2 = null`, `operator = null`），同时解除除零、溢出或负数开方导致的锁定状态。
  3. **退格撤销 (DEL / Backspace)**：移除当前正在输入的数值的最后一个字符。若当前数值长度被清减为 0，则该数值重置为 `0`。
  4. **触屏可达性保障**：通过按键的大小、对比度及间距设计，确保在移动端设备上，`C`、`AC`、`DEL` 等控制按键具备足够的点击面积（不小于 44px * 44px）及防误触保护。
- **输入输出数据流描述**：
  - **输入**：清除/重置/退格指令。
  - **内部处理**：重置内存中对应的状态变量或对当前操作数执行字符串切片（`slice(0, -1)`）。
  - **输出**：更新界面显示区的值（如重置为 `0` 或缩减后的数字字符串）。

#### REQ-CALC-003: 辅助运算功能 (Decimal / Percent / Negation)
- **优先级**：中
- **前置条件**：系统加载成功。
- **业务逻辑**：
  1. **小数点输入 (.)**：在当前操作数末尾添加小数点。若当前操作数已存在小数点，则此按键无效（防止输入如 `1.2.3`）。若当前未输入任何数字时点击 `.`, 则自动补全为 `0.`。
  2. **百分比转换 (%)**：将当前操作数的值除以 100（即 `val = val / 100`）。
  3. **正负号切换 (+/-)**：反转当前操作数的正负符号（即 `val = -val`）。
- **输入输出数据流描述**：
  - **输入**：小数点/百分比/正负号按键指令。
  - **内部处理**：对当前操作数（字符串或数值）进行正则校验或数值转换。
  - **输出**：更新界面显示。

#### REQ-CALC-004: 边界、异常与溢出处理规则
- **优先级**：高
- **前置条件**：计算动作被触发或用户正在输入。
- **业务逻辑**：
  1. **除零防护**：若运算符为 `/` 且 `operand2` 的值为 `0`，当触发 `=` 或链式计算时，系统不可崩溃，必须在显示区输出 `Error` 字符。此时锁定除 `AC` 键外的所有按键，系统仅允许通过 `AC` 按键进行复位解除锁定。
  2. **数值溢出与异常防护**：
     - **科学计数法显示**：统一为：当计算结果绝对值 `>= 1e15` (即 `abs(result) >= 1e15`) 时使用科学计数法，系统必须自动将计算结果转换为科学计数法格式（如 `1.23456789e+15`）进行展示，以防止界面排版溢出。
     - **最大安全整数溢出锁定**：当计算结果或输入数值的绝对值超过 JavaScript 的最大安全整数限制（即大于 `Number.MAX_SAFE_INTEGER` [9007199254740991] 或小于 `Number.MIN_SAFE_INTEGER` [-9007199254740991]）时，界面显示区必须显示 `Error: Overflow`。此时系统进入强行锁定状态，禁用除 `AC` 键外的所有按键，且【仅允许】点击 `AC` 键进行清空、复位及解除锁定。
  3. **高精度纠错**：使用高精度处理（运算后先通过 `toFixed(10)` 转换，再使用 `parseFloat` 解析），确保最多保留 10 位有效小数，避免 JavaScript 语言原生浮点数计算精度丢失（如 `0.1 + 0.2 === 0.30000000000000004`）的问题。
  4. **输入长度限制**：统一单次操作数的最大输入长度上限为 12 位字符，且明确此 12 位限制【包含】负号（`-`）和小数点（`.`）。一旦输入字符串达到 12 位，系统将忽略后续任何数字或小数点输入指令。
- **输入输出数据流描述**：
  - **输入**：计算操作数及运算符。
  - **内部处理**：校验除数、验证数值安全边界、转换科学计数法、纠正浮点精度、校验字符长度。
  - **输出**：格式化后的数值、`Error` 或 `Error: Overflow` 状态文本。

#### REQ-CALC-005: 历史记录管理
- **优先级**：中
- **前置条件**：系统加载成功。
- **业务逻辑**：
  1. **记录写入**：每次触发 `=` 计算并获得有效结果时，系统自动将完整的计算公式和计算结果（形如 `2.5 + 4 = 6.5`）封装为单条历史记录，同步写入 `Session Storage`（会话态存储），保证页面刷新不丢失。此外，当触发平方或开方运算且获得有效非报错结果时，同样自动将计算公式和计算结果（形如 `sqr(5) = 25` 或 `sqrt(9) = 3`）封装为单条历史记录并同步写入 Session Storage。
  2. **记录查看**：界面提供一个可切换隐藏/显示的“历史记录”侧边栏或独立面板，将存储中的历史记录以时间倒序的列表形式渲染。
  3. **记录回填行为**：用户点击历史记录列表中的任意一条记录时，系统必须【仅以该历史记录的计算结果作为 operand1 继续后续计算】。在点击回填的瞬间，系统需自动【清空当前正在输入的任何表达式、运算符以及 operand2】，将该历史计算结果直接载入并替换为当前唯一的待操作数 `operand1`，使计算器进入以该历史结果为基础、等待用户输入下一个运算符的就绪状态。
  4. **记录清空与二次确认**：历史面板底部提供“清空历史”按钮。为防范用户误触，系统必须强制实施【二次确认步骤】：用户点击清空时，系统必须弹出带有明确“确认”和“取消/关闭”选项的二次确认对话框（“确定要清除所有计算历史吗？”）。只有当用户显式点击“确认”后，方可清除 `Session Storage` 中的历史记录并重置历史列表；若点击取消或关闭弹窗，则不执行任何清除操作。
  5. **空状态展示**：当会话中没有任何历史计算记录时，历史面板应展示清晰友好的空状态（如展示灰色占位符及提示文本：“无历史记录”）。
- **输入输出数据流描述**：
  - **输入**：计算结算事件、点击历史条目事件、清空并确认事件。
  - **内部处理**：对 `Session Storage` 进行 `setItem` / `getItem` / `clear` 操作，以及状态机的操作数回填。
  - **输出**：历史列表内容更新、弹窗反馈、计算器显示区内容变更为历史结果。

#### REQ-CALC-006: 平方运算 (Square)
- **优先级**：中
- **前置条件**：系统加载成功，当前显示有有效的数值（`operand1` 或正在输入的 `operand2`）。
- **业务逻辑**：
  1. 用户在输入完数值后，点击平方按键（$x^2$），系统立即对当前正在编辑或显示的数值进行平方运算（$result = val \times val$）。
  2. 若当前正处于 `operand1` 输入状态，平方计算结果将直接替换 `operand1`；若处于 `operand2` 输入状态，平方计算结果将直接替换 `operand2`。
  3. 平方计算后，计算结果显示在屏幕上，且系统状态机保持当前运算就绪状态。
  4. 计算过程同样适用 REQ-CALC-004 定义的数值溢出与高精度纠错规则。
  5. 历史记录同步：计算成功且未报错时，系统自动将该单值平方运算（形如 `sqr(5) = 25`）作为一条记录同步写入 Session Storage。
- **输入输出数据流描述**：
  - **输入**：当前操作数 $val$（数值）。
  - **内部处理**：执行 $val \times val$，校验安全整数限制、应用高精度纠偏。
  - **输出**：平方计算结果。
- **验收标准**：
  - **AC-CALC-006-1**：用户输入数字 `8`，点击平方按键，屏幕即刻显示 `64`。
  - **AC-CALC-006-2**：用户输入负数 `-5`，点击平方按键，屏幕即刻显示 `25`。
  - **AC-CALC-006-3**：若输入数值平方计算结果的绝对值超过 `Number.MAX_SAFE_INTEGER` 时，屏幕显示 `Error: Overflow`，且系统锁定，仅允许 AC 按键复位。
  - **AC-CALC-006-4**：输入浮点数 `0.3`，点击平方按键，屏幕显示 `0.09`，不发生浮点精度丢失。

#### REQ-CALC-007: 开方运算 (Square Root)
- **优先级**：中
- **前置条件**：系统加载成功，当前显示有有效的数值（`operand1` 或正在输入的 `operand2`）。
- **业务逻辑**：
  1. 用户在输入完数值后，点击开方按键（$\sqrt{x}$），系统立即对当前正在编辑或显示的数值进行开平方运算（$result = \sqrt{val}$）。
  2. 若当前正处于 `operand1` 输入状态，开方计算结果将直接替换 `operand1`；若处于 `operand2` 输入状态，开方计算结果将直接替换 `operand2`。
  3. 开方计算后，计算结果显示在屏幕上，且系统状态机保持当前运算就绪状态。
  4. **负数开方校验**：若当前操作数为负数，点击开方按键后系统必须在屏幕上显示 `Error`，锁定除 `AC` 键外的所有按键，仅允许通过 `AC` 按键复位。
  5. 计算过程同样适用 REQ-CALC-004 定义的数值溢出与高精度纠错规则。
  6. 历史记录同步：计算成功且未报错时，系统自动将该单值开方运算（形如 `sqrt(9) = 3`）作为一条记录同步写入 Session Storage。
- **输入输出数据流描述**：
  - **输入**：当前操作数 $val$（数值）。
  - **内部处理**：校验 $val$ 是否小于 0；若非负，则执行 $\sqrt{val}$ 计算并应用高精度纠偏。
  - **输出**：开方计算结果或异常信息 `Error`。
- **验收标准**：
  - **AC-CALC-007-1**：用户输入数字 `16`，点击开方按键，屏幕即刻显示 `4`。
  - **AC-CALC-007-2**：用户输入数字 `-9`，点击开方按键，屏幕显示 `Error`，且除 AC 键外所有按键被锁定。
  - **AC-CALC-007-3**：输入数字 `2`，点击开方按键，屏幕显示 `1.4142135624`（保留 10 位有效小数）。
  - **AC-CALC-007-4**：输入数字 `0`，点击开方按键，屏幕显示 `0`。

### 3.2 界面与适配模块 (UI)

#### REQ-UI-001: 响应式跨平台适配
- **优先级**：高
- **前置条件**：无。
- **业务逻辑**：
  1. 采用 CSS Grid (网格布局) 设计计算器键盘，确保所有按键分布均匀。
  2. 针对 PC、平板和手机使用媒体查询 (Media Queries) 实施响应式布局适配，严格采用以下屏幕分辨率断点：
     - **手机端 (窄屏, <480px)**：计算器占满手机屏幕高度与宽度（全屏模式），按键触控面积符合移动端触控标准（建议不小于 44px * 44px）。
     - **平板端 (中屏, 480px 至 768px)**：计算器渲染为独立的居中卡片布局，按键触控面积按比例放大，保证舒适的操作体验。
     - **PC端 (宽屏, >768px)**：计算器渲染为独立的居中浮动卡片（最大宽度限制为 400px），并配有现代化阴影和圆角效果。
- **输入输出数据流描述**：
  - **输入**：客户端设备屏幕分辨率信息。
  - **内部处理**：CSS 媒体查询按预设断点动态应用不同的样式类（卡片布局与全屏布局的自动切换）。
  - **输出**：完美的响应式视觉渲染，无任何内容溢出或按钮错位。

#### REQ-UI-002: 现代美学设计、明暗主题切换与持久化
- **优先级**：中
- **前置条件**：系统运行中。
- **业务逻辑**：
  1. 采用现代极简主义设计风格，为数字键、特殊操作符键和运算结果显示区设计不同的配色体系，确保对比度高且易于识别。
  2. **主题跟随系统**：系统加载时，默认使用 CSS 媒体查询 `@media (prefers-color-scheme: dark)` 侦测操作系统的主题首选项，自动适配为对应的明色或暗色主题。
  3. **用户手动切换与 LocalStorage 持久化**：提供主题切换控件，支持一键在 **Light Mode (日间明亮主题)** 与 **Dark Mode (夜间暗色主题)** 之间进行流畅过渡。一旦用户手动切换主题，系统应使用 `LocalStorage` 持久化记录用户的主题偏好（键名为 `theme`）。后续访问时，系统优先读取此持久化首选项。
  4. 为所有按键实现悬停（Hover）及点击（Active）的背景微色变和轻微缩放过渡动效，提升交互的实物感。
- **输入输出数据流描述**：
  - **输入**：主页加载、主题切换开关触发事件。
  - **内部处理**：读取/写入 `LocalStorage` 状态，动态更新 HTML 标签的 `data-theme` 属性值或改变全局 CSS 变量。
  - **输出**：计算器视觉风格切换，并在 300ms 内流畅渐变完成。

#### REQ-UI-003: 物理键盘映射与快捷键
- **优先级**：中
- **前置条件**：PC 端环境加载成功。
- **业务逻辑**：
  1. **物理键盘映射支持**：系统需绑定键盘事件，正式将 `%` 键、`Delete` 键、`c` 键与 `C` 键纳入物理快捷键映射集合中。具体映射集合定义如下：
     - 数字 `0-9` 与小数点 `.`：映射为数字与小数点输入。
     - `+`、`-`、`*`、`/`：映射为对应的加、减、乘、除运算指令。
     - `Enter` 或 `=`：映射为触发计算求值（`=`）。
     - `Backspace` 和 `Delete`：均映射为删除当前操作数的末位字符（`DEL`）。
     - `Escape`：映射为全部清除（`AC`）。
     - `%`：映射为百分比运算。
     - `c` 与 `C` 键：当前清除快捷键统一为同时支持小写 `c` 与大写 `C` 映射，触发清除当前输入（C）功能。
     - **平方与开方**：物理键盘暂不为平方和开方运算提供快捷键映射。
  2. **物理按键 Active 反馈时序**：统一为：当用户按下已被映射的物理按键时，屏幕对应的虚拟按键触发 `.active` 状态并在按压期间持续，直到 KeyUp 时恢复原状（即 `.active` 状态在按压期间持续，直到 KeyUp 时恢复）。此 Active 状态视觉效果（CSS 类名需为 `.active`）须与鼠标/触屏点击时完全一致，以提供实时的按键视觉反馈。
  3. **快捷键说明面板**：PC 端界面显著位置（如计算器顶部角落）提供一个“快捷键说明”入口（通常为 `?` 图标或小字按钮）。点击该入口开启面板或模态窗口后，必须展示如下【具体文字定义】的快捷键映射关系：“快捷键说明：数字 0-9 键、小数点 . 键输入数字；+ - * / 键触发运算；Enter 或 = 键触发计算；Backspace / Delete 键删除末位；Escape 键全部清除（AC）；% 键触发百分比运算；c 或 C 键清除当前输入（C）”。
- **输入输出数据流描述**：
  - **输入**：物理键盘按下的 `KeyDown` 事件。
  - **内部处理**：拦截并解析 `event.key`，执行对应运算逻辑，并动态为对应的虚拟按键添加激活类（Active State Class）。
  - **输出**：虚拟按键高亮动效触发，运算状态及结果展示同步更新。

#### REQ-UI-004: 结果快捷复制与 Toast 提示
- **优先级**：中
- **前置条件**：计算结果显示区有内容展示。
- **业务逻辑**：
  1. **一键复制结果**：用户用手指或鼠标点击计算结果显示区域时，系统应自动执行复制操作，将当前显示的数值或科学计数法结果写入客户端系统的剪贴板中。
  2. **Toast 提示显示**：复制动作成功完成后，界面显著位置应即刻弹出轻量级的非阻塞 Toast 提示框，其【反馈文案必须为】：“结果已复制到剪贴板”。
  3. **Toast 渐变淡出与生命周期**：Toast 提示框的展示应遵循严格的【2秒生命周期】与【渐变淡出行为】：提示框渲染展示后保持 2 秒钟（2000ms）的静止显示，随后在 300ms 内通过 CSS 透明度（opacity）平滑过渡到 0，执行完淡出动画后，对应的 DOM 节点必须被销毁。
- **输入输出数据流描述**：
  - **输入**：显示区域的点击事件。
  - **内部处理**：使用 HTML5 Clipboard API (`navigator.clipboard.writeText`) 操作剪贴板，并触发 Toast 计时器。
  - **输出**：文本写入剪贴板，Toast 提示框在屏幕上方/中央渲染展示，并在 2 秒后淡出消失。

# 软件需求说明书 (SRS) — 简易 Web 计算器 (simple-web-calculator)

## 1. 系统边界与概述 (System Boundary & Overview)
- **业务概述**：设计并实现一个运行在 Web 端的简易计算器，兼顾美观性与良好的跨平台适配性，满足用户日常的快捷计算需求。
- **系统边界**：
  - **在册范围**：
    - 计算器核心计算引擎：基础加减乘除、清除（C/AC）、退格（Backspace）、百分比、正负号切换、小数点输入、平方运算（x²）、开方运算（√）、幂运算（^）、常用对数运算（log）、自然对数运算（ln）、自定义底数对数运算（log_y(x)）、倒数运算（1/x）、三角函数运算（sin、cos、tan）、内置圆周率常数（π）以及答案调用（ans）功能。
    - 响应式网页界面：自适应 PC、平板及手机等不同分辨率的屏幕。
    - 视觉美化与交互体验：提供悬停与点击动效，支持明暗（Dark & Light）双色主题切换，并包含用于三角函数计算的 Deg/Rad 单位切换模式。
    - 计算安全与边界控制：浮点精度校正、除零溢出校验、数值溢出控制、输入字符长度限制。
    - 辅助实用功能：会话态历史记录管理、物理键盘快捷键映射及快捷键说明、结果复制与 Toast 提示。
  - **排除范围**：
    - 除三角函数（sin、cos、tan）和对数、幂运算、倒数、常数/存储外的其他高等数学/统计学高级计算功能（如双曲函数、微积分、复数计算、矩阵运算等）。
    - 历史计算记录的云端同步或数据库持久化存储（本需求仅支持前端会话态的历史查看）。
    - 第三方账户体系或社交分享功能。

## 2. 数据流拓扑 (Data Flow Topology)
- **数据流描述**：
  计算器接收用户的按键输入（数字/运算符/单值科学计算键），触发相应的数据模型校验，存入内部状态机（操作数1、运算符、操作数2）。触发 `=` 或单值科学计算键时执行运算引擎处理，校正精度，并最终输出更新至前端显示屏。若遇到异常（如除零、安全整数溢出、负数开方、非法幂运算、非正数对数运算），显示屏直接输出异常状态词并执行状态锁保护。计算成功的结果会同步写入 Session Storage 历史记录中，并可通过点击回填。
- **核心数据流向图/流程**：
```

  [用户按键输入/键盘快捷键] -> (输入有效性校验/长度限制) -> [更新内存操作数与运算符]
       |
       v (触发 "=" 计算或更改运算符)
  [执行计算引擎] -> (判断除数是否为零/是否安全整数溢出) 
       |---> [是] -> [显示 "Error" / "Error: Overflow"，锁定输入，仅允许 AC 清除]
       |---> [否] -> (值绝对值 >= 1e15 ? 转换科学计数法 : 执行高精度纠偏 toFixed(10)) 
                          |
                          v
                    [更新显示结果，更新当前操作数] -> [写入 Session Storage 历史记录]

```

## 3. 功能需求定义 (Functional Requirements)

### 3.1 计算引擎模块 (CALC)

#### REQ-CALC-001: 基础四则运算
- **优先级**：高
- **前置条件**：系统加载成功，用户已输入第一个操作数和运算符，且正在输入第二个操作数。
- **业务逻辑**：
  1. 用户点击数字或小数点，构造 `operand1`。
  2. 用户点击运算符（`+`, `-`, `*`, `/`），将状态机的当前运算符设为所选符，并将输入焦点切换至 `operand2`。
  3. 用户点击 `=` 或另一个运算符时，系统根据当前的两个操作数与运算符计算出结果 `result`。
  4. 若计算结果为有效数值，将 `result` 更新为新的 `operand1`，并清空 `operand2` 与运算符，以便进行链式计算。
- **输入输出数据流描述**：
  - **输入**：`operand1` (数字), `operator` (字符 `+`, `-`, `*`, `/`), `operand2` (数字)。
  - **内部处理**：根据 `operator` 对 `operand1` 和 `operand2` 执行标准的数学运算。
  - **输出**：计算结果 `result`，并在界面显示区呈现。

#### REQ-CALC-002: 清除与撤销机制 (C / AC / Backspace)
- **优先级**：高
- **前置条件**：系统加载成功，界面存在输入内容。
- **业务逻辑**：
  1. **清除当前输入 (C)**：重置当前正在输入的数值（如 `operand2` 或当前的 `operand1`），不影响已输入的运算符及另一个操作数。
  2. **全部清除 (AC)**：将整个计算器的状态机重置为初始状态（`operand1 = 0`, `operand2 = null`, `operator = null`），同时解除除零、溢出或负数开方导致的锁定状态。
  3. **退格撤销 (DEL / Backspace)**：移除当前正在输入的数值的最后一个字符。若当前数值长度被清减为 0，则该数值重置为 `0`。
  4. **触屏可达性保障**：通过按键的大小、对比度及间距设计，确保在移动端设备上，`C`、`AC`、`DEL` 等控制按键具备足够的点击面积（不小于 44px * 44px）及防误触保护。
- **输入输出数据流描述**：
  - **输入**：清除/重置/退格指令。
  - **内部处理**：重置内存中对应的状态变量或对当前操作数执行字符串切片（`slice(0, -1)`）。
  - **输出**：更新界面显示区的值（如重置为 `0` 或缩减后的数字字符串）。

#### REQ-CALC-003: 辅助运算功能 (Decimal / Percent / Negation)
- **优先级**：中
- **前置条件**：系统加载成功。
- **业务逻辑**：
  1. **小数点输入 (.)**：在当前操作数末尾添加小数点。若当前操作数已存在小数点，则此按键无效（防止输入如 `1.2.3`）。若当前未输入任何数字时点击 `.`, 则自动补全为 `0.`。
  2. **百分比转换 (%)**：将当前操作数的值除以 100（即 `val = val / 100`）。
  3. **正负号切换 (+/-)**：反转当前操作数的正负符号（即 `val = -val`）。
- **输入输出数据流描述**：
  - **输入**：小数点/百分比/正负号按键指令。
  - **内部处理**：对当前操作数（字符串或数值）进行正则校验或数值转换。
  - **输出**：更新界面显示。

#### REQ-CALC-004: 边界、异常与溢出处理规则
- **优先级**：高
- **前置条件**：计算动作被触发或用户正在输入。
- **业务逻辑**：
  1. **除零防护**：若运算符为 `/` 且 `operand2` 的值为 `0`，当触发 `=` 或链式计算时，系统不可崩溃，必须在显示区输出 `Error` 字符。此时锁定除 `AC` 键外的所有按键，系统仅允许通过 `AC` 按键进行复位解除锁定。
  2. **数值溢出与异常防护**：
     - **科学计数法显示**：统一为：当计算结果绝对值 `>= 1e15` (即 `abs(result) >= 1e15`) 时使用科学计数法，系统必须自动将计算结果转换为科学计数法格式（如 `1.23456789e+15`）进行展示，以防止界面排版溢出。
     - **最大安全整数溢出锁定**：当计算结果或输入数值的绝对值超过 JavaScript 的最大安全整数限制（即大于 `Number.MAX_SAFE_INTEGER` [9007199254740991] 或小于 `Number.MIN_SAFE_INTEGER` [-9007199254740991]）时，界面显示区必须显示 `Error: Overflow`。此时系统进入强行锁定状态，禁用除 `AC` 键外的所有按键，且【仅允许】点击 `AC` 键进行清空、复位及解除锁定。
  3. **高精度纠错**：使用高精度处理（运算后先通过 `toFixed(10)` 转换，再使用 `parseFloat` 解析），确保最多保留 10 位有效小数，避免 JavaScript 语言原生浮点数计算精度丢失（如 `0.1 + 0.2 === 0.30000000000000004`）的问题。
  4. **输入长度限制**：统一单次操作数的最大输入长度上限为 12 位字符，且明确此 12 位限制【包含】负号（`-`）和小数点（`.`）。一旦输入字符串达到 12 位，系统将忽略后续任何数字或小数点输入指令。
- **输入输出数据流描述**：
  - **输入**：计算操作数及运算符。
  - **内部处理**：校验除数、验证数值安全边界、转换科学计数法、纠正浮点精度、校验字符长度。
  - **输出**：格式化后的数值、`Error` 或 `Error: Overflow` 状态文本。

#### REQ-CALC-005: 历史记录管理
- **优先级**：中
- **前置条件**：系统加载成功。
- **业务逻辑**：
  1. **记录写入**：每次触发 `=` 计算并获得有效结果时，系统自动将完整的计算公式和计算结果（形如 `2.5 + 4 = 6.5`）封装为单条历史记录，同步写入 `Session Storage`（会话态存储），保证页面刷新不丢失。此外，当触发平方、开方、对数运算、倒数运算、三角函数运算且获得有效非报错结果时，同样自动将计算公式和计算结果（形如 `sqr(5) = 25`、`sqrt(9) = 3`、`1/(5) = 0.2`、`sin(30) = 0.5`、`log(100) = 2`、`ln(e) = 1` 或 `log(2)(8) = 3`）封装为单条历史记录并同步写入 Session Storage。
  2. **记录查看**：界面提供一个可切换隐藏/显示的“历史记录”侧边栏或独立面板，将存储中的历史记录以时间倒序的列表形式渲染。
  3. **记录回填行为**：用户点击历史记录列表中的任意一条记录时，系统必须【仅以该历史记录的计算结果作为 operand1 继续后续计算】。在点击回填的瞬间，系统需自动【清空当前正在输入的任何表达式、运算符以及 operand2】，将该历史计算结果直接载入并替换为当前唯一的待操作数 `operand1`，使计算器进入以该历史结果为基础、等待用户输入下一个运算符的就绪状态。
  4. **记录清空与二次确认**：历史面板底部提供“清空历史”按钮。为防范用户误触，系统必须强制实施【二次确认步骤】：用户点击清空时，系统必须弹出带有明确“确认”和“取消/关闭”选项的二次确认对话框（“确定要清除所有计算历史吗？”）。只有当用户显式点击“确认”后，方可清除 `Session Storage` 中的历史记录并重置历史列表；若点击取消或关闭弹窗，则不执行任何清除操作。
  5. **空状态展示**：当会话中没有任何历史计算记录时，历史面板应展示清晰友好的空状态（如展示灰色占位符及提示文本：“无历史记录”）。
- **输入输出数据流描述**：
  - **输入**：计算结算事件、点击历史条目事件、清空并确认事件。
  - **内部处理**：对 `Session Storage` 进行 `setItem` / `getItem` / `clear` 操作，以及状态机的操作数回填。
  - **输出**：历史列表内容更新、弹窗反馈、计算器显示区内容变更为历史结果。

#### REQ-CALC-006: 平方运算 (Square)
- **优先级**：中
- **前置条件**：系统加载成功，当前显示有有效的数值（`operand1` 或正在输入的 `operand2`）。
- **业务逻辑**：
  1. 用户在输入完数值后，点击平方按键（$x^2$），系统立即对当前正在编辑或显示的数值进行平方运算（$result = val \times val$）。
  2. 若当前正处于 `operand1` 输入状态，平方计算结果将直接替换 `operand1`；若处于 `operand2` 输入状态，平方计算结果将直接替换 `operand2`。
  3. 平方计算后，计算结果显示在屏幕上，且系统状态机保持当前运算就绪状态。
  4. 计算过程同样适用 REQ-CALC-004 定义的数值溢出与高精度纠错规则。
  5. 历史记录同步：计算成功且未报错时，系统自动将该单值平方运算（形如 `sqr(5) = 25`）作为一条记录同步写入 Session Storage。
- **输入输出数据流描述**：
  - **输入**：当前操作数 $val$（数值）。
  - **内部处理**：执行 $val \times val$，校验安全整数限制、应用高精度纠偏。
  - **输出**：平方计算结果。
- **验收标准**：
  - **AC-CALC-006-1**：用户输入数字 `8`，点击平方按键，屏幕即刻显示 `64`。
  - **AC-CALC-006-2**：用户输入负数 `-5`，点击平方按键，屏幕即刻显示 `25`。
  - **AC-CALC-006-3**：若输入数值平方计算结果的绝对值超过 `Number.MAX_SAFE_INTEGER` 时，屏幕显示 `Error: Overflow`，且系统锁定，仅允许 AC 按键复位。
  - **AC-CALC-006-4**：输入浮点数 `0.3`，点击平方按键，屏幕显示 `0.09`，不发生浮点精度丢失。

#### REQ-CALC-007: 开方运算 (Square Root)
- **优先级**：中
- **前置条件**：系统加载成功，当前显示有有效的数值（`operand1` 或正在输入的 `operand2`）。
- **业务逻辑**：
  1. 用户在输入完数值后，点击开方按键（$\sqrt{x}$），系统立即对当前正在编辑或显示的数值进行开平方运算（$result = \sqrt{val}$）。
  2. 若当前正处于 `operand1` 输入状态，开方计算结果将直接替换 `operand1`；若处于 `operand2` 输入状态，开方计算结果将直接替换 `operand2`。
  3. 开方计算后，计算结果显示在屏幕上，且系统状态机保持当前运算就绪状态。
  4. **负数开方校验**：若当前操作数为负数，点击开方按键后系统必须在屏幕上显示 `Error`，锁定除 `AC` 键外的所有按键，仅允许通过 `AC` 按键复位。
  5. 计算过程同样适用 REQ-CALC-004 定义的数值溢出与高精度纠错规则。
  6. 历史记录同步：计算成功且未报错时，系统自动将该单值开方运算（形如 `sqrt(9) = 3`）作为一条记录同步写入 Session Storage。
- **输入输出数据流描述**：
  - **输入**：当前操作数 $val$（数值）。
  - **内部处理**：校验 $val$ 是否小于 0；若非负，则执行 $\sqrt{val}$ 计算并应用高精度纠偏。
  - **输出**：开方计算结果或异常信息 `Error`。
- **验收标准**：
  - **AC-CALC-007-1**：用户输入数字 `16`，点击开方按键，屏幕即刻显示 `4`。
  - **AC-CALC-007-2**：用户输入数字 `-9`，点击开方按键，屏幕显示 `Error`，且除 AC 键外所有按键被锁定。
  - **AC-CALC-007-3**：输入数字 `2`，点击开方按键，屏幕显示 `1.4142135624`（保留 10 位有效小数）。
  - **AC-CALC-007-4**：输入数字 `0`，点击开方按键，屏幕显示 `0`。

#### REQ-CALC-008: 幂运算 (Exponentiation)
- **优先级**：中
- **前置条件**：系统加载成功，用户已输入第一个操作数（基数 `operand1`）并选择幂运算运算符 `^`，且正在输入第二个操作数（指数 `operand2`）。
- **业务逻辑**：
  1. **双值运算机制**：用户输入基数 `operand1`，点击 `^` 键，将状态机当前运算符设为 `^`，然后输入指数 `operand2`。当用户点击 `=` 或其他运算符时触发幂运算（$result = operand1 ^ {operand2}$）。
  2. **输入范围与异常校验**：
     - **负底数限制**：若基数 `operand1` 为负数，则指数 `operand2` 必须为整数。若指数 `operand2` 为小数（非整数），系统必须输出 `Error`，并强制进入锁定状态（除 `AC` 键外，禁用并锁定其他所有按键）。
     - **零底数限制**：若基数 `operand1` 为零（`0`），则指数 `operand2` 必须大于或等于零。若指数 `operand2` 小于零（负数），系统必须输出 `Error`（代表除零异常），并强制进入锁定状态（除 `AC` 键外，禁用并锁定其他所有按键）。
  3. **计算与溢出处理**：
     - 若幂运算结果的绝对值超过最大安全整数限制 `9007199254740991`，系统必须在显示区输出 `Error: Overflow`，并强制进入锁定状态（除 `AC` 键外，禁用并锁定其他所有按键）。
     - 计算过程同样适用 REQ-CALC-004 定义的科学计数法显示、高精度纠错等规则。
  4. **历史记录同步**：计算成功且未报错时，系统自动将该幂运算（形如 `2 ^ 3 = 8`）作为一条记录同步写入 Session Storage。
- **输入输出数据流描述**：
  - **输入**：基数 `operand1` (数值), 运算符 `^` (字符), 指数 `operand2` (数值)。
  - **内部处理**：校验基数与指数的合法组合；若合法，执行 $operand1^{operand2}$ 计算，校验安全整数范围并执行精度纠偏。
  - **输出**：幂运算结果、`Error` 或 `Error: Overflow` 异常文本。
- **验收标准**：
  - **AC-CALC-008-1**：用户输入数字 `2`，点击 `^` 按键，输入数字 `3`，点击 `=`，屏幕即刻显示 `8`。
  - **AC-CALC-008-2**：用户输入数字 `-2`，点击 `^` 按键，输入 `0.5`，点击 `=`，屏幕显示 `Error`，且除 AC 键外所有按键被锁定。
  - **AC-CALC-008-3**：用户输入数字 `0`，点击 `^` 按键，输入 `-1`，点击 `=`，屏幕显示 `Error`，且除 AC 键外所有按键被锁定。
  - **AC-CALC-008-4**：用户输入数字 `2`，点击 `^` 按键，输入 `60`，点击 `=`，计算结果超出最大安全整数，屏幕显示 `Error: Overflow` 并锁定。
  - **AC-CALC-008-5**：用户输入数字 `2`，点击 `^` 按键，输入 `-3`，点击 `=`，屏幕显示 `0.125`，不发生浮点精度丢失。

#### REQ-CALC-009: 对数运算 (Logarithm - ln, log, log_y)
- **优先级**：中
- **前置条件**：系统加载成功，当前显示有有效的数值（`operand1` 或正在输入的 `operand2`）。
- **业务逻辑**：
  1. **常用对数 log 运算机制**：单值运算。用户在输入完数值后，点击常用对数按键（`log`），系统立即对当前正在编辑或显示的数值进行以 10 为底的对数运算（$result = \log_{10}(val)$）。
  2. **自然对数 ln 运算机制**：单值运算。用户在输入完数值后，点击自然对数按键（`ln`），系统立即对当前正在编辑或显示的数值进行以自然常数 $e$ 为底的对数运算（$result = \ln(val)$）。
  3. **自定义底数对数 log_y(x) 运算机制**：双值运算。用户输入底数 $y$，点击自定义底数对数按键（`log_y`），系统将状态机当前运算符设为 `log_y`，然后用户输入真数 $x$。当点击 `=` 键或其他运算符时，系统执行自定义底数对数运算（$result = \log_y(x) = \frac{\ln(x)}{\ln(y)}$）。
  4. **输入范围与异常校验**：
     - **单值对数限制**：进行 `log` 或 `ln` 运算时，当前操作数 $val$ 必须满足 $val > 0$。若 $val \le 0$，系统在点击对数键后立即在显示屏输出 `Error`，并强制进入锁定状态（除 `AC` 键外，禁用并锁定其他所有按键），仅允许通过 `AC` 键复位。
     - **自定义底数对数限制**：
       - 底数 $y$（即 `operand1`）必须满足 $y > 0$ 且 $y \ne 1$。若 $y \le 0$ 或 $y = 1$，点击对数键或触发求值时，系统必须输出 `Error` 并强制进入锁定状态。
       - 真数 $x$（即 `operand2`）必须满足 $x > 0$。若 $x \le 0$，触发求值时，系统必须输出 `Error` 并强制进入锁定状态。
  5. **计算与溢出处理**：
     - 若对数运算结果的绝对值超过最大安全整数限制 `9007199254740991`，系统必须显示 `Error: Overflow`，并强制进入锁定状态（除 `AC` 键外，禁用并锁定其他所有按键）。
     - 计算过程同样适用 REQ-CALC-004 定义的科学计数法显示、高精度纠错等规则。
  6. **历史记录同步**：计算成功且未报错时，系统自动将对应的对数运算公式和结果写入 Session Storage：
     - 常用对数：形如 `log(100) = 2`。
     - 自然对数：形如 `ln(2.7182818284) = 1`。
     - 自定义底数对数：形如 `log(2)(8) = 3` 或 `log_2(8) = 3`。
- **输入输出数据流描述**：
  - **输入**：对数类型（log/ln/log_y），操作数值。
  - **内部处理**：校验底数和真数的合法性，执行对数计算并应用高精度纠偏。
  - **输出**：对数计算结果、`Error` 或 `Error: Overflow` 异常文本。
- **验收标准**：
  - **AC-CALC-009-1**：用户输入数字 `100`，点击 `log` 按键，屏幕即刻显示 `2`。
  - **AC-CALC-009-2**：用户输入数字 `2.7182818284`，点击 `ln` 按键，屏幕显示 `1`。
  - **AC-CALC-009-3**：用户输入数字 `0` 或负数（如 `-5`），点击 `log` 或 `ln` 按键，屏幕显示 `Error`，且除 AC 键外所有按键被锁定。
  - **AC-CALC-009-4**：用户输入底数 `2`，点击 `log_y` 按键，输入真数 `8`，点击 `=`，屏幕即刻显示 `3`。
  - **AC-CALC-009-5**：用户输入底数 `1`，点击 `log_y` 按键，输入真数 `5`，点击 `=`，屏幕显示 `Error`，且除 AC 键外所有按键被锁定。
  - **AC-CALC-009-6**：用户输入底数 `2`，点击 `log_y` 按键，输入真数 `0` 或负数（如 `-4`），点击 `=`，屏幕显示 `Error` 并进入锁定状态。

#### REQ-CALC-010: 倒数运算 (Reciprocal)
- **优先级**：中
- **前置条件**：系统加载成功，当前显示有有效的数值（`operand1` 或正在输入的 `operand2`）。
- **业务逻辑**：
  1. **单值运算机制**：用户在输入完数值后，点击倒数按键（`1/x`），系统立即对当前正在编辑或显示的数值进行倒数运算（$result = 1 / val$）。
  2. **分母零值限制与锁定**：输入值 $val$ 不能为 0。若当前操作数为零（`0`），点击倒数按键后，系统必须在显示屏输出 `Error`，并强制进入锁定状态（除 `AC` 键外，禁用并锁定其他所有按键），仅允许通过 `AC` 按键进行复位以解除锁定。
  3. **计算与溢出处理**：
     - 若倒数运算结果的绝对值超过最大安全整数限制 `9007199254740991`，系统必须显示 `Error: Overflow`，并强制进入锁定状态（除 `AC` 键外，禁用并锁定其他所有按键）。
     - 计算过程同样适用 REQ-CALC-004 定义的科学计数法显示、高精度纠错等规则。
  4. **历史记录同步**：计算成功且未报错时，系统自动将该单值倒数运算（形如 `1/(5) = 0.2`）作为一条记录同步写入 Session Storage。
- **输入输出数据流描述**：
  - **输入**：当前操作数 $val$（数值）。
  - **内部处理**：校验 $val$ 是否为 0；若不为 0，执行 $1/val$ 计算，校验安全整数限制并执行高精度纠偏。
  - **输出**：倒数计算结果、`Error` 或 `Error: Overflow` 异常文本。
- **验收标准**：
  - **AC-CALC-010-1**：用户输入数字 `5`，点击倒数按键，屏幕即刻显示 `0.2`。
  - **AC-CALC-010-2**：用户输入数字 `0`，点击倒数按键，屏幕显示 `Error`，且除 AC 键外所有按键被锁定。
  - **AC-CALC-010-3**：用户输入极小值（如 `1e-16`），点击倒数按键，计算结果超过最大安全整数范围，屏幕显示 `Error: Overflow` 且除 AC 键外所有按键被锁定。
  - **AC-CALC-010-4**：用户输入数字 `-4`，点击倒数按键，屏幕显示 `-0.25`。

#### REQ-CALC-011: 三角函数运算 (Trigonometric Functions)
- **优先级**：中
- **前置条件**：系统加载成功，当前显示有有效的数值（`operand1` 或正在输入的 `operand2`）。
- **业务逻辑**：
  1. **单值运算机制**：用户在输入完数值后，点击对应的三角函数按键（`sin`、`cos`、`tan`），系统立即对当前正在编辑或显示的数值进行三角函数计算。
  2. **角度制与弧度制模式（Deg/Rad）**：
     - 系统默认使用**角度制（Deg）**进行计算。
     - 界面上需提供一个 `Deg/Rad` 切换控制按键，点击可在角度制（Deg）与弧度制（Rad）模式之间切换，且不改变当前计算器屏幕中已显示的操作数。
     - **角度制（Deg）模式下**：系统将输入的数值视为角度值 $x_{deg}$，计算时需将其转换为弧度值再执行运算，计算公式为：$\sin(x_{deg} \times \pi / 180)$。
     - **弧度制（Rad）模式下**：系统直接将输入的数值视为弧度值 $x_{rad}$ 并执行运算。
  3. **正切函数（tan）无定义点限制与锁定**：
     - 在角度制（Deg）下，若输入数值 $x$ 为 $90 + 180k$（$k$ 为任意整数）度时，正切函数无定义。
     - 在弧度制（Rad）下，若输入数值 $x$ 为 $\pi/2 + k\pi$（$k$ 为任意整数）弧度时，正切函数无定义。
     - **异常校验**：若用户对无定义点的值点击 `tan`，系统必须在屏幕上显示 `Error`，并强制进入锁定状态（除 `AC` 键外，禁用并锁定其他所有按键）。由于浮点数存在精度限制，系统在检测 `tan` 输入时，若发现分母接近于 0（即 $\cos(x_{rad})$ 绝对值极小，如小于 $10^{-10}$），均必须触发该 `Error` 锁定逻辑。
  4. **高精度微差纠错**：
     - 由于 JavaScript 原生三角函数运算和 `Math.PI` 近似精度的限制，某些标准值运算会产生微差（如在角度制下 `sin(30)` 理论上为 `0.5`，`cos(90)` 为 `0`）。
     - 系统在进行三角函数运算后，必须对这些典型值执行微差纠偏处理，确保例如 `sin(30)` 结果准确显示为 `0.5`，`cos(90)` 结果准确显示为 `0`，`tan(45)` 结果准确显示为 `1`。
  5. **历史记录同步**：计算成功且未报错时，系统自动将该三角函数运算（形如 `sin(30) = 0.5` 或 `cos(1.0471975512) = 0.5`）作为一条记录同步写入 Session Storage。
- **输入输出数据流描述**：
  - **输入**：当前操作数 $val$（数值）、所选函数（`sin`/`cos`/`tan`）及当前三角函数计算模式（Deg/Rad）。
  - **内部处理**：根据计算模式换算单位，校验正切无定义边界，调用三角函数计算并执行高精度微差纠偏。
  - **输出**：三角计算结果或异常信息 `Error`。
- **验收标准**：
  - **AC-CALC-011-1**：在默认 Deg 模式下，输入数字 `30` 并点击 `sin`，屏幕即刻显示 `0.5`。
  - **AC-CALC-011-2**：在默认 Deg 模式下，输入数字 `90` 并点击 `cos`，屏幕显示 `0`（确保纠偏生效，避免显示微小浮点误差如 `6.12323e-17`）。
  - **AC-CALC-011-3**：在默认 Deg 模式下，输入数字 `90` 并点击 `tan`，屏幕显示 `Error` 并进入锁定状态。
  - **AC-CALC-011-4**：切换到 Rad 模式，输入数字 `3.1415926536` 并点击 `sin`，屏幕显示 `0`。
  - **AC-CALC-011-5**：用户点击主题或切换 Deg/Rad 时，计算器内已显示的数字和当前的计算运算符不受干扰。

#### REQ-CALC-012: 常数与存储 (Constants & Memory Ans)
- **优先级**：中
- **前置条件**：系统加载成功。
- **业务逻辑**：
  1. **内置圆周率常数 π**：
     - 界面需提供 `π` 按键。
     - 当用户点击 `π` 键时，若当前正处于等待新操作数输入状态，系统直接将当前操作数替换为 $\pi$ 的近似值 `3.141592653589793`；若用户正在编辑数字，点击 `π` 将丢弃当前正在编辑的数字并直接替换为该 $\pi$ 近意值。
     - 圆周率常数的值显示在屏幕上时，需根据 REQ-CALC-004 的 12 位输入长度限制，截断显示为 `3.14159265359`（包含负号或小数点共 12 位）。
  2. **内置答案调用功能 ans**：
     - 系统状态机需在内存中维护一个 `ans` 全局变量，用于持久化（在当前页面会话内）保存【最近一次成功计算所得的有效非报错数值结果】。若计算结果为 `Error` 或 `Error: Overflow`，则不对 `ans` 进行更新。
     - `ans` 变量的初始值在页面首次加载或执行 `AC` 全部清除时重置为 `0`。
     - 界面需提供 `ans` 按键。当用户点击 `ans` 键时，系统将当前正在输入的数值或操作数直接替换为当前 `ans` 变量所存储的值。
  3. **计算连贯性**：
     - 点击 `π` 键或 `ans` 键回填的数值，等同于用户手动输入的常规数字，可以直接进行后续的四则运算、平方、开方、对数、三角函数及倒数等运算。
- **输入输出数据流描述**：
  - **输入**：点击 `π` 键或点击 `ans` 键。
  - **内部处理**：获取 `Math.PI` 或 `ans` 的当前值，根据最大长度限制截断后载入到当前操作数。
  - **输出**：在屏幕显示区呈现对应常数值。
- **验收标准**：
  - **AC-CALC-012-1**：用户点击 `π` 键，屏幕即刻显示 `3.14159265359`。
  - **AC-CALC-012-2**：计算 `5 * 2 =`，屏幕显示 `10`。随后点击 `C` 或输入 `ans`，再次点击 `+ 3 =`，屏幕显示 `13`，证明 `ans` 正确存储并调用了上一次成功计算的结果 `10`。
  - **AC-CALC-012-3**：初次加载页面，点击 `ans`，屏幕显示 `0`；进行 `AC` 清除后，点击 `ans`，屏幕仍显示 `0`。
  - **AC-CALC-012-4**：计算 `5 / 0 =`，屏幕显示 `Error`。随后用户点击 `AC` 清除异常，并输入 `ans`，屏幕显示 `0`（若在 `5 / 0` 之前没有计算过其他值，则为 `0`；若之前有成功的计算结果如 `10`，则 `ans` 仍保持 `10`，不被 `Error` 覆盖）。
  - **AC-CALC-012-5**：点击 `π` 键，随后点击平方 `x²` 按键，屏幕显示 `9.8696044011`。

- **优先级**：高

#### REQ-UI-003: 物理键盘映射与快捷键

#### REQ-UI-004: 结果快捷复制与 Toast 提示
- **优先级**：中
- **前置条件**：计算结果显示区有内容展示。
- **业务逻辑**：
  1. **一键复制结果**：用户用手指或鼠标点击计算结果显示区域时，系统应自动执行复制操作，将当前显示的数值或科学计数法结果写入客户端系统的剪贴板中。
  2. **Toast 提示显示**：复制动作成功完成后，界面显著位置应即刻弹出轻量级的非阻塞 Toast 提示框，其【反馈文案必须为】：“结果已复制到剪贴板”。
  3. **Toast 渐变淡出与生命周期**：Toast 提示框的展示应遵循严格的【2秒生命周期】与【渐变淡出行为】：提示框渲染展示后保持 2 秒钟（2000ms）的静止显示，随后在 300ms 内通过 CSS 透明度（opacity）平滑过渡到 0，执行完淡出动画后，对应的 DOM 节点必须被销毁。
- **输入输出数据流描述**：
  - **输入**：显示区域的点击事件。
  - **内部处理**：使用 HTML5 Clipboard API (`navigator.clipboard.writeText`) 操作剪贴板，并触发 Toast 计时器。
  - **输出**：文本写入剪贴板，Toast 提示框在屏幕上方/中央渲染展示，并在 2 秒后淡出消失。

#### REQ-UI-005: iPad 横竖屏切换适配
- **优先级**：中
- **前置条件**：系统在 iPad 设备或平板端浏览器上运行，且检测到屏幕方向发生旋转切换。
- **业务逻辑**：
  1. **屏幕方向检测**：系统需通过 CSS 媒体查询（`orientation: portrait` 与 `orientation: landscape`）及 JavaScript 视口改变事件，动态检测并响应 iPad 设备的横竖屏状态。
  2. **竖屏模式（Portrait）排版适配**：
     - **按键布局**：采用标准的垂直网格布局（如 4 列或 5 列网格），按键间距保持适中，确保单手握持时的主要操作按键（如数字、基础运算符）易于触达。
     - **显示区域**：计算结果显示屏高度占计算器容器总高度的 20% 至 25%，字符大小自适应，避免长数值溢出。
     - **内容显示变化**：为了在窄屏下保持界面简洁，历史记录侧边栏默认隐藏，用户必须通过点击顶部历史图标手动唤出，唤出时以半透明遮罩层覆盖在计算器主体之上。
  3. **横屏模式（Landscape）排版适配**：
     - **按键布局**：利用宽屏空间，布局切换为扁平化的横向宽网格。部分功能键（如科学计算键或历史记录控制键）可与主按键区并排展示，以降低按键的垂直高度比例，防止拉伸变形。且 iPad 横屏模式下，虚拟按键的尺寸（width/height）及间距（margin/gap）需等比增加 20%，以提升指尖触控体验。
     - **显示区域**：显示屏高度占比调整为 15% 至 20%，横向宽度拉伸，且字符根据视口宽度自适应调整字号。
     - **内容显示变化**：由于横屏空间充足，系统默认将“历史记录”侧边栏（包含历史记录列表及清空历史按钮）在侧边（如左侧或右侧）展开展示，无需手动点击唤出，且历史记录面板不得遮挡或压盖主计算按键区，但允许用户通过点击按钮进行折叠/收起。
  4. **旋转过渡动效**：横竖屏切换时，按键的大小、位置重排及历史记录面板的折叠/展开、常驻/隐藏状态切换，系统需应用平滑的渐变或平移动效，过渡时间控制在 200ms 至 300ms 之间，以保证视觉连贯性。
- **输入输出数据流描述**：
  - **输入**：客户端设备发送的屏幕方向改变事件（`orientationchange` 或 `resize`）及视口 CSS 媒介查询状态。
  - **内部处理**：根据检测到的屏幕方向（横屏或竖屏），激活对应的样式类，调整网格布局模板（Grid Template）、重置历史面板的展示状态，并动态计算显示屏的自适应字号。
  - **输出**：计算器界面流畅重排。竖屏下历史记录默认隐藏并提供悬浮遮罩；横屏下历史记录默认在侧边展开并可折叠/收起，按键布局合理且尺寸与间距等比增加 20% 无形变，过渡耗时 200ms 至 300ms。
- **验收标准**：
  - **AC-UI-005-1**：iPad 在竖屏（Portrait）方向下，计算器键盘呈现垂直网格，历史记录侧边栏默认不显示；点击历史图标后，历史记录面板以遮罩层形式悬浮在计算器主体上方。
  - **AC-UI-005-2**：将 iPad 旋转至横屏（Landscape）方向后，界面在 300ms 内平滑重排为宽屏布局，历史记录面板默认在侧边展开展示，且不重叠、不压盖任何计算器按键，用户可点击按钮将其折叠/收起。
  - **AC-UI-005-3**：无论是横屏还是竖屏模式，在 12 位最大输入长度上限内，输入或计算结果的字体大小均能随视口自动缩放，保证数值在单行完整显示，不发生换行或部分截断。
  - **AC-UI-005-4**：iPad 横竖屏切换后，所有虚拟按键的点击/触控感应区域面积不得小于 44px * 44px，且切换过程无闪烁或错位现象。

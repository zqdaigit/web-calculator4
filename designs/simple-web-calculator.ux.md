# 交互设计说明书: 简易 Web 计算器 (simple-web-calculator)

## 1. 业务目标与用户画像
* **目标**：提供一个界面美观、操作流畅、支持自适应布局（手机、平板、PC）的网页版基础计算器，帮助用户快速进行日常数学运算，并可查看历史计算记录。
* **核心用户**：需要在各种设备（移动端/桌面端）上进行快速计算的普通网页访问用户。

## 2. 页面跳转流程 (User Flow)
* **计算器主页流程**：
  - 用户打开页面 -> 展示默认计算器界面（显示 `0`，历史记录为空）。
  - 用户点击键盘按键/物理键盘输入 -> 实时显示运算表达式与当前结果。
  - 用户点击等号按键 `[ = ]` 或按回车键 -> 计算并输出最终结果，同时将本条算式记录推入历史记录。
* **清除与重置交互**：
  - 用户点击 `[ C ]` 按键 -> 清空当前正在输入的数值，不重置已输入的运算符及另一个操作数。
  - 用户点击 `[ AC ]` 按键 -> 将计算器状态完全重置为初始状态，恢复显示为 `0`。
  - 用户点击 `[ ⌫ ]` 按键 -> 回退删除当前操作数的最后一位字符。
* **除零异常流与恢复**：
  - 当发生除以零的错误时（如输入 `5 / 0 =`） -> 界面输出错误提示且状态锁定（除 `AC` 键外，锁定其他所有按键） -> 用户必须点击 `[ AC ]` 键重置状态以恢复计算功能。
* **历史记录交互**：
  - **手机端 (<480px)**：点击顶部 `[ 📜 历史记录 ]` 按钮 -> 自底部向上滑出历史记录抽屉。点击某条历史算式 -> 该算式及结果重新载入当前计算器，抽屉自动收起。
  - **平板 (480px~768px) 及 PC 端 (>768px)**：计算器右侧包含折叠式的历史记录面板。点击顶部 `[ 📜 历史 ]` 按钮 -> 向右展开历史面板（宽度增加）。点击面板中的历史记录 -> 重新载入计算器。点击 `[ 🗑️ 清空历史 ]` -> 清除本地历史并呈现空状态。

## 4. 交互细节与状态说明 (Interaction Details)

### 4.1 按键与控制面板交互
* **数字按键 (0-9)**：
  - **默认状态**：高亮文字底色，悬停时底色变深，点击时有微缩放动效和阴影反馈。
  - **输入限制**：单次操作数最大长度限制为 12 位字符（包含数字和小数点），达到上限后按键指令被忽略。
* **运算符按键 (+, -, *, /, %)**：
  - **交互响应**：点击后将当前运算符加入表达式。如果连续输入多个运算符，后者覆盖前者（例如输入 `+` 再输入 `*`，变更为 `*`）。
* **功能按键**：
  - **[ AC ] (All Clear)**：重置全部计算状态（操作数1、操作数2、运算符），显示区域恢复为 `0`，解锁受限状态。
  - **[ C ] (Clear)**：仅清除当前正在输入的数值（如 `operand2`），不重置已输入的运算符及另一个操作数。
  - **[ ⌫ ] (退格/Backspace)**：在主显示区右下角提供该触控按钮。点击可删除当前操作数的最后一位字符。若数值长度被清减为 0，则重置显示为 `0`。
  - **[ +/- ] (正负切换)**：反转当前操作数的正负符号。
  - **[ . ] (小数点)**：当前操作数无小数点时可输入；已包含小数点时点击无响应。若当前无数字时点击 `.`, 自动补全为 `0.`。
  - **[ = ] (等号)**：触发计算，更新显示结果并推入历史记录。
* **键盘快捷键映射**：
  - 数字键 `0-9` 映射对应数字。
  - `+` `-` `*` `/` `%` 映射对应运算。
  - `.` 映射小数点。
  - `Enter` 或 `=` 映射等号触发计算。
  - `Backspace` 或 `Delete` 映射退格键，删除最后一位输入字符。
  - `Escape` 映射全局清除键 `AC`，`c` 映射当前清除键 `C`。

### 4.2 显示区域设计
* **表达式显示区 (上层)**：
  - 字号较小，字体颜色较浅（如灰色），显示当前正在输入的完整算式（如 `123 + 456`）。
* **主结果显示区 (下层)**：
  - 字号较大，高亮颜色，显示当前输入的数字、异常状态或最终结果。
  - 字体采用等宽数字字体，确保数字排列整齐。
  - 支持点击该区域复制内容到剪贴板，复制成功时在顶部展示 Toast 提示 `"已复制到剪贴板"`。

### 4.3 主题切换 (暗黑模式)
* **默认状态**：跟随系统主题或默认亮色模式。
* **切换响应**：点击右上角主题切换按钮，页面整体色彩以渐变过渡（过渡时间为 0.3s）切换至暗黑模式，并记住用户选择。

## 5. 异常流与兜底设计 (Exception Flows)
* **除数为零 (Division by Zero)**：
  - **触发场景**：输入例如 `5 / 0 =`
  - **展现方式**：主显示区显示 `Error`。
  - **恢复机制**：除 `AC` 键外，锁定并禁用计算器的其他所有按键。用户必须点击 `[ AC ]` 键清除全部状态以恢复正常的按键交互。
* **数值溢出 (Overflow / Underflow)**：
  - **触发场景**：计算结果超出 `1e12` 或数值超出 JavaScript 最大安全数限制。
  - **展现方式**：若结果绝对值大于 `1e12`，采用科学计数法显示（如 `1.234567e+12`）。若超出安全数限制，主显示区显示 `Error: Overflow`。
* **浮点数运算精度修正**：
  - **触发场景**：进行浮点数运算。
  - **设计策略**：统一将浮点计算结果精度限制在 10 位有效小数以内（如 `0.1 + 0.2` 显示为 `0.3`），防止浮点数精度丢失问题导致不良体验。
* **历史记录空状态**：
  - **展示效果**：无历史记录时，历史区域显示文字 `"暂无历史计算记录"`，图标淡化展示。

## 6. HTML 原型实现约束 (HTML Prototype Constraints)
* **状态切换**：历史记录抽屉（Mobile 端）及历史侧栏（Tablet/PC 端）需使用 CSS transition 模拟滑出/收起效果。
* **假数据与动态计算**：静态原型应具备真正的 JavaScript 基础计算逻辑，无需后端交互。
* **样式自适应**：必须使用 CSS Media Queries（以手机 `<480px`，平板 `480px-768px`，PC `>768px` 为断点）实现无缝自适应切换。

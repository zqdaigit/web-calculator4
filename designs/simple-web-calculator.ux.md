# 交互设计说明书: 简易 Web 计算器 (simple-web-calculator)

## 1. 业务目标与用户画像
* **目标**：提供一个界面美观、操作流畅、支持自适应布局（手机、平板、PC）的网页版基础计算器，帮助用户快速进行日常数学运算，并可查看历史计算记录。
* **核心用户**：需要在各种设备（移动端/桌面端）上进行快速计算的普通网页访问用户。

## 2. 页面跳转流程 (User Flow)
* **计算器主页流程**：
  - 用户打开页面 -> 展示默认计算器界面（默认显示 0）
  - 用户点击/按键输入 -> 实时显示运算表达式与当前结果 -> 点击 `=` 或按回车键 -> 计算并输出最终结果，同时记录到历史列表
* **历史记录交互**：
  - 手机端：点击 [历史记录] 按钮 -> 自底部或侧边滑出历史记录抽屉 -> 点击某条历史记录 -> 该记录的算式和结果重新载入计算器输入框 -> 点击关闭或空白处回退主界面。
  - 平板/PC端：历史记录常驻右侧面板 -> 点击某条历史记录 -> 重新载入计算器 -> 点击历史面板上的 [清空] 按钮 -> 弹窗确认后清空历史记录。

## 4. 交互细节与状态说明 (Interaction Details)

### 4.1 按键与控制面板交互
* **数字按键 (0-9)**：
  - **默认状态**：高亮文字底色，支持悬停 (Hover) 变深，点击 (Active) 时有微弱缩放动画及阴影反馈。
  - **输入限制**：单词输入最大长度限制为 15 位数字，超出后按键无响应，避免显示溢出。
* **运算符按键 (+, -, *, /, %)**：
  - **交互响应**：点击后将当前运算符加入表达式。如果连续输入多个运算符，后者覆盖前者（例如输入 `+` 再输入 `*`，变更为 `*`）。
* **功能按键**：
  - **[ C ] (Clear)**：点击清除所有输入，显示恢复为 `0`。
  - **[ +/- ] (正负切换)**：对当前输入数字进行正负号取反。
  - **[ . ] (小数点)**：当前数字无小数点时可输入；已包含小数点时点击无响应。
  - **[ = ] (等号)**：触发计算，更新显示结果并推入历史记录。
* **键盘快捷键映射**：
  - 数字键 0-9 映射对应数字。
  - `+` `-` `*` `/` `%` 映射对应运算。
  - `.` 映射小数点。
  - `Enter` 或 `=` 映射等号触发计算。
  - `Backspace` 或 `Delete` 映射退格键，删除最后一位输入字符。
  - `Escape` 映射清除键 `C`。

### 4.2 显示区域设计
* **表达式显示区 (上层)**：
  - 字号较小，字体颜色较浅（如灰色），显示当前正在输入的完整算式（如 `123 + 456`）。
* **主结果显示区 (下层)**：
  - 字号较大，高亮颜色，显示当前输入的数字或最终结果。
  - 字体采用等宽数字字体，确保数字排列整齐。
  - 支持点击该区域复制内容到剪贴板，复制成功时在顶部展示 Toast 提示 `"已复制到剪贴板"`。

### 4.3 主题切换 (暗黑模式)
* **默认状态**：跟随系统主题或默认亮色模式。
* **切换响应**：点击右上角主题切换按钮，页面整体色彩以渐变过渡（过渡时间为 0.3s）切换至暗黑模式，并记住用户选择。

## 5. 异常流与兜底设计 (Exception Flows)
* **除数为零 (Division by Zero)**：
  - **触发场景**：输入例如 `5 / 0 =`
  - **展现方式**：主显示区显示 `错误: 除数不能为零`（或英文显示 `Error: Div by 0`），所有运算符按钮置灰禁用，必须点击 `C` 后才可重新使用。
* **数值溢出 (Overflow / Underflow)**：
  - **触发场景**：计算结果超出计算极限或精度过小。
  - **展现方式**：若结果绝对值大于 `1e15`，采用科学计数法显示（如 `1.234567e+16`）。若数值超出 JavaScript 最大安全数限制，主显示区显示 `Error: Overflow`。
* **浮点数运算精度修正**：
  - **触发场景**：进行浮点数运算（如 `0.1 + 0.2`）。
  - **设计策略**：统一将计算结果限制在 12 位有效数字以内，使用精度控制函数处理（如显示 `0.3` 而非 `0.30000000000000004`），防止产生体验缺陷。
* **历史记录空状态**：
  - **展示效果**：无历史记录时，历史区域显示文字 `"暂无历史计算记录"`，图标淡化展示。

## 6. HTML 原型实现约束 (HTML Prototype Constraints)
* **状态切换**：历史记录抽屉（Mobile 端）需使用 CSS transition 模拟滑出/收起效果。
* **假数据与动态计算**：静态原型应具备真正的 JavaScript 基础计算逻辑，无需后端交互。
* **样式自适应**：必须使用 CSS Media Queries（或 Tailwind CSS 响应式类名）实现无缝切换。

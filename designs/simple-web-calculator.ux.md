# 交互设计说明书: 简易 Web 计算器 (simple-web-calculator)

## 1. 业务目标与用户画像
* **目标**：提供一个界面美观、操作流畅、支持自适应布局（手机、平板、PC）的网页版基础计算器，帮助用户快速进行日常数学运算，并可查看历史计算记录。
* **核心用户**：需要在各种设备（移动端/桌面端）上进行快速计算的普通网页访问用户。

## 2. 页面跳转流程 (User Flow)
* **计算器主页流程**：
  - 用户打开页面 -> 展示默认计算器界面（显示 `0`，历史记录为空）。
  - 用户点击键盘按键/物理键盘输入 -> 实时显示运算表达式与当前结果。
  - 用户点击等号按键 `[ = ]` 或按回车键 -> 计算并输出最终结果，同时将本条算式记录推入历史记录。
* **清除与重置交互**：
  - 用户点击 `[ C ]` 按键 -> 清空当前正在输入的数值，不重置已输入的运算符及另一个操作数。
  - 用户点击 `[ AC ]` 按键 -> 将计算器状态完全重置为初始状态，恢复显示为 `0`，解除一切溢出或除零导致的按键锁定状态。
  - 用户点击 `[ ⌫ ]` 按键 -> 回退删除当前操作数的最后一位字符。
* **除零与溢出异常流锁定与恢复**：
  - 当发生除以零错误（如输入 `5 / 0 =`）或数值溢出（主显示区显示 `Error` 或 `Error: Overflow`）时 -> 界面进入异常锁定状态（除 `AC` 键外，锁定并禁用其他所有按键） -> 用户必须点击 `[ AC ]` 键重置状态以恢复计算功能。
* **历史记录交互**：
  - **手机端 (<480px)**：点击顶部 `[ 📜 历史记录 ]` 按钮 -> 自底部向上滑出历史记录抽屉。点击某条历史记录 -> **仅提取其计算结果**回填至当前计算器（作为当前操作数 `operand1`），抽屉自动收起，用户可以此结果为基准继续进行下一轮计算。
  - **平板 (480px~768px) 及 PC 端 (>768px)**：计算器右侧包含折叠式的历史记录面板。点击顶部 `[ 📜 历史 ]` 按钮 -> 向右展开历史面板（卡片宽度增加）。点击面板中的某条历史记录 -> **仅提取其计算结果**回填至当前计算器中。点击 `[ 🗑️ 清空历史 ]` -> 弹出二次确认弹窗 -> 确认后清空历史并呈现空状态。
* **iPad 横竖屏适配交互**：
  - **状态变化响应**：当 iPad 检测到屏幕旋转时，页面整体布局应在 300ms 内平滑调整以适应新的屏幕宽高比。
  - **竖屏 (Portrait)**：布局逻辑同手机端或小型平板，计算器区域居中，历史记录面板通过点击按钮滑出（类似于手机端抽屉式）。
  - **横屏 (Landscape)**：布局逻辑同 PC 端，计算器主体位于左侧/居中，历史记录面板默认侧展开（或保持可折叠状态，且宽度更宽以展示更多历史信息）。按键区域大小及间距根据横屏空间自动按比例拉伸/缩放，确保大屏幕下手指点击的舒适度。

## 4. 交互细节与状态说明 (Interaction Details)

### 4.1 按键与控制面板交互
* **数字按键 (0-9)**：
  - **默认状态**：高亮文字底色，悬停时底色变深，点击时有微缩放动效和阴影反馈。
  - **输入限制**：单次操作数最大长度限制为 12 位字符（包含数字、小数点和正负号 `-`），达到上限后任何新输入的数字或小数点按键指令都将被忽略。
* **运算符按键 (+, -, *, /, %)**：
  - **交互响应**：点击后将当前运算符加入表达式。如果连续输入多个运算符，后者覆盖前者（例如输入 `+` 再输入 `*`，变更为 `*`）。
* **科学计算按键 (x², √, ^, log)**：
  - **[ x² ] (平方)**：计算当前操作数的平方。
  - **[ √ ] (开方)**：计算当前操作数的平方根。
  - **[ ^ ] (幂运算)**：触发幂运算模式。
  - **[ log ] (对数)**：计算以 10 为底的对数。
* **功能按键**：
  - **[ AC ] (All Clear)**：重置全部计算状态（操作数1、操作数2、运算符），显示区域恢复为 `0`，解除一切除零或溢出导致的锁定状态。
  - **[ C ] (Clear)**：仅清除当前正在输入的数值（如 `operand2`），不重置已输入的运算符及另一个操作数。
  - **[ ⌫ ] (退格/Backspace)**：在主显示区右下角提供该触控按钮。点击可删除当前操作数的最后一位字符。若数值长度被清减为 0，则重置显示为 `0`。
  - **[ +/- ] (正负切换)**：反转当前操作数的正负符号。
  - **[ . ] (小数点)**：当前操作数无小数点时可输入；已包含小数点时点击无响应。若当前无数字时点击 `.`, 自动补全为 `0.`。
  - **[ = ] (等号)**：触发计算，更新显示结果并推入历史记录。
* **物理键盘快捷键映射与交互反馈**：
  - 支持以下物理按键与虚拟按键的映射：
    | 物理按键 | 对应虚拟按键 / 功能描述 |
    | :--- | :--- |
    | `0` - `9` | `[ 0 ]` - `[ 9 ]` 数字输入 |
    | `.` | `[ . ]` 小数点 |
    | `+` | `[ + ]` 加法 |
    | `-` | `[ - ]` 减法 |
    | `*` | `[ * ]` 乘法 |
    | `/` | `[ / ]` 除法 |
    | `%` | `[ % ]` 百分比转换 |
    | `Enter` 或 `=` | `[ = ]` 确认计算求值 |
    | `Backspace` 或 `Delete` | `[ ⌫ ]` 退格删除最后一位字符 |
    | `Escape` | `[ AC ]` 全部重置清除并解锁状态 |
    | `c` 或 `C` | `[ C ]` 清除当前操作数 |
  - **虚拟按键视觉反馈**：当用户在物理键盘上按下上述映射按键时，界面上相对应的虚拟按键需同步触发激活态动效（添加 `.active` 类，背景色加深，并伴随按压位移/微缩放触感反馈）。该激活状态在物理按键被按压期间持续显示，直至按键被抬起 (KeyUp) 时即刻恢复原状，以实现时序完全一致的手键合一感官反馈。
  - **快捷键说明面板**：PC端右上角常驻 `[ ⌨️ 快捷键说明 ]` 入口。点击后弹出半透明黑色遮罩层（Opacity 0.5）以及中心居中的悬浮模态框（Modal）。模态框标题为“物理键盘快捷键指南”，以清晰表格列出上述物理键盘映射表，右下角配有 `[ 关闭 ]` 按钮，同时支持点击遮罩空白区域自动关闭。
* **iPad 横竖屏适配交互增强**：
  - **按键动态缩放**：iPad 在横屏模式下，虚拟按键的容器宽度随屏幕宽度等比放大，按键大小 (width/height) 与边距 (margin/gap) 增加 20%，确保指尖触控体验提升。
  - **过渡动画**：从竖屏旋转到横屏时，所有 UI 元素执行 0.3s 的 `ease-in-out` 过渡效果，避免突兀的布局跳变。

### 4.2 显示区域设计
* **表达式显示区 (上层)**：
  - 字号较小，字体颜色较浅（如灰色），显示当前正在输入的完整算式（如 `123 + 456`）。
* **主结果显示区 (下层)**：
  - 字号较大，高亮颜色，显示当前输入的数字、异常状态或最终结果.
  - 字体采用等宽数字字体，确保数字排列整齐。
  - **一键复制结果与 Toast 提示**：支持点击该区域执行复制。点击后自动将当前显示的数值或科学计数法结果复制入剪贴板。复制成功后，即刻在计算器屏幕中上部弹出轻量非阻塞黑色半透明 Toast 提示，提示内容为：“结果已复制到剪贴板”。Toast 提示展示 2 秒后，通过渐变动画（Opacity 在 300ms 内由 1 降为 0）自动淡出并销毁。

### 4.3 主题切换 (暗黑模式)
* **默认状态**：跟随系统主题或默认亮色模式。
* **切换响应**：点击右上角主题切换按钮，页面整体色彩以渐变过渡（过渡时间为 0.3s）切换至暗黑模式，并记住用户选择。

### 4.4 历史记录列表及其操作
* **历史计算回填**：点击历史记录列表中的任意一条记录，系统**仅提取**该条记录的**计算结果**，回填加载为当前操作数（即替换为 `operand1`，而不要将整串旧公式拉回编辑状态），以便用户以此结果为基础直接进行下一次运算。
* **清空历史与二次确认**：点击历史面板底部 `[ 🗑️ 清空历史 ]` 按钮，弹出二次确认弹窗对话框。
  - 对话框文案：“确定要清除所有计算历史吗？”
  - 交互选项：`[ 确认 ]`（警示红底白字，点击清除 Session Storage 中的历史记录并更新界面，关闭对话框）与 `[ 取消 ]`（浅灰底黑字，点击直接关闭对话框而不执行清除）。
* **历史空状态**：若没有历史记录，展示淡色图标与文字提示：“无历史记录”。

## 5. 异常流与兜底设计 (Exception Flows)
* **除数为零 (Division by Zero)**：
  - **触发场景**：输入例如 `5 / 0 =`
  - **展现方式**：主显示区显示 `Error`。
  - **恢复机制**：系统强制进入锁定状态，除 `AC` 键外锁定并禁用其他所有按键。用户必须点击 `[ AC ]` 键清空状态，以恢复按键交互功能。
* **数值溢出 (Overflow / Underflow)**：
  - **触发场景**：计算结果超出安全数值边界。
  - **科学计数法转换**：当计算结果的绝对值大于等于 `1e15`（即 `abs(result) >= 1e15`）时，系统自动将计算结果转换为科学计数法格式展示（如 `1.234567e+15`），以防止显示溢出。
  - **安全整数溢出锁定**：若计算结果或输入数值超出 JavaScript 的最大/最小安全整数范围（绝对值大于最大安全整数 `9007199254740991`），主显示区显示 `Error: Overflow`。此时系统强制进入锁定状态，除 `AC` 键外，禁用并锁定其他所有虚拟按键。只有点击 `[ AC ]` 键清空状态后才可解除锁定，恢复按键交互。
* **浮点数运算精度修正**：
  - **触发场景**：进行浮点数运算。
  - **设计策略**：统一将浮点计算结果精度限制在 10 位有效小数以内（如 `0.1 + 0.2` 显示为 `0.3`），防止浮点数精度丢失问题导致不良体验。
* **历史记录空状态**：
  - **展示效果**：无历史记录时，历史区域显示文字 `"无历史记录"`，图标淡化展示。

## 6. HTML 原型实现约束 (HTML Prototype Constraints)
* **状态切换**：历史记录抽屉（Mobile 端）及历史侧栏（Tablet/PC 端）需使用 CSS transition 模拟滑出/收起效果。
* **假数据与动态计算**：静态原型应具备真正的 JavaScript 基础计算逻辑，无需后端交互。
* **样式自适应**：必须使用 CSS Media Queries（以手机 `<480px`，平板 `480px-768px`，PC `>768px` 为断点）实现无缝自适应切换。
* **旋转适配**：需在 `window.onorientationchange` 或 `resize` 事件中动态检测并触发 UI 重绘，确保 iPad 横竖屏切换时布局平滑过渡。
